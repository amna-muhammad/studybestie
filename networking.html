🌐 How Networking Powers StudyBestie

This page explains how networking technologies make the StudyBestie website accessible on the internet.

1. Domain Name System (DNS)

When a user types the website address into the browser, the Domain Name System (DNS) converts the domain name into an IP address. This allows the browser to locate the server that hosts the StudyBestie website.

Example command used to find the IP address of the domain: 

nslookup yourdomain.com

The DNS server responds with the IP address where the website is hosted.

2. IP Addressing

Every website server has a unique Internet Protocol (IP) address. When DNS resolves the domain name of StudyBestie, it returns an IP address that identifies the server hosting the website.

Example: 

Domain: studybestie.example.com
IPv4 Address: 185.199.108.153

The browser uses this IP address to send requests to the correct web server.

3. HTTP vs HTTPS

When a user opens the StudyBestie website, the browser communicates with the web server using the HTTP protocol. Modern websites use HTTPS, which encrypts communication between the browser and server using TLS.

Example request shown in browser developer tools: 

Request URL: https://studybestie.example.com
Status Code: 200 OK
Protocol: HTTPS

The status code 200 OK means the server successfully returned the webpage.

4. TCP/IP Communication

Web browsers communicate with servers using the TCP/IP protocol suite. IP handles addressing and routing packets across the internet, while TCP ensures reliable delivery of data.

When a user visits StudyBestie:

  • The browser sends a request to the server's IP address.
  • The request travels across networks using TCP/IP.
  • The server processes the request.
  • The server sends the webpage back to the browser.

Summary

Networking technologies such as DNS, IP addressing, HTTPS, and TCP/IP make it possible for users to access the StudyBestie website from anywhere on the internet. These systems translate domain names into IP addresses, securely transmit data, and ensure reliable communication between browsers and servers.
